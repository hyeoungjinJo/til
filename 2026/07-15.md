# 2026-07-15

- 목록/배열 반환으로 시그니처를 바꾼 메서드는 mock 스텁도 배열 반환에 맞춰 갱신해야 테스트가 깨지지 않는다.
- 대량 삭제는 개별 delete 반복 대신 bulk delete로 전환하면 성능·쿼리 수가 줄고, 부팅 시 자동 시드 호출은 운영에서 비활성화해 불필요한 초기화를 막는다.
- 비밀번호 재설정 시 동일 연락처로 묶인 계정 목록을 반환해, 한 연락처에 여러 계정이 연결된 케이스를 사용자가 선택하게 한다.
- 스냅샷 기반 리포트·정산에서 원본이 없는 경우 null을 흡수(기본값 처리)해 조회 실패를 막는다.
- 콘텐츠 소스가 여러 개일 때(예: 로컬 데이터 + 외부 CMS) 목록 응답 단계에서 병합하고, 세션/결과 생성 시 소스별로 분기 처리한다. 스냅샷 컬럼 + ContentMode 플래그 + 외부 콘텐츠 클라이언트 조합으로 소스를 추상화한다.
- 리소스의 다국어는 언어별 슬롯 구조로 관리한다. 신규 생성·기존(legacy) 수정 양쪽 경로 모두에서 슬롯을 반영하고, 목록 응답에도 슬롯을 포함시켜야 언어 데이터 누락이 없다.
- 외부 엔진(Unity 등) 연동은 리소스 풀 연동 규약을 문서로 남겨 백엔드-클라이언트 계약을 고정한다.
